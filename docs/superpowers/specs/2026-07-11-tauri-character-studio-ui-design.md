# Tauri 角色工作室 UI 修复设计

## 背景

提交 `ae9885c` 为 Tauri 设置页新增了两个角色工作室入口，并在独立工作室中实现了角色列表页和一套简化的配色界面。当前实现存在三类问题：

- 设置页的“编辑当前角色”和“打开角色工作室”作用重复。
- 工作室通过单独的角色列表页切换角色，当前编辑对象不能始终被看见。
- 工作室使用 Unicode 导航符号、独立主题字段文案和简化色块控件，和主设置页的视觉及交互不一致。

## 目标

- 设置页只保留一个明确的角色编辑入口，并放在当前角色下拉菜单旁边。
- 工作室始终显示当前编辑角色，允许直接切换已有角色或新建角色。
- 工作室的导航、主题字段、配色控件和主题预览与主设置页保持一致。
- 保存角色不会改变主程序当前使用的角色。
- 切换角色或新建角色时不静默丢弃未保存修改。

## 非目标

- 不合并设置程序和角色工作室进程。
- 不改变角色包格式、角色注册表格式或当前角色持久化规则。
- 不新增角色删除、复制、发布或在线角色管理功能。
- 不重构两个 Tauri 前端为新的通用构建系统。

## 设置页入口

“当前角色”设置行右侧改为一个横向控件组：

- 左侧继续使用现有 `characterSelect` 下拉菜单。
- 右侧放置一个“修改角色”次级按钮。
- 按钮使用下拉菜单当前选中的角色 ID 调用 `studio.launch`。
- 没有可用角色或角色包操作进行中时，按钮禁用。

删除独立的“角色工作室”设置组，以及 `characterStudioCurrentButton`、`characterStudioOpenButton` 两套事件处理。后端 `studio.launch` RPC 保持单一入口，不新增第二种启动协议。

## 工作室角色选择

工作室移除“角色列表”导航项、角色列表页面、搜索框和刷新按钮。详情页页头下方增加持久工具栏：

- “当前编辑角色”下拉菜单始终可见。
- 下拉菜单右侧放置“新建角色”按钮。
- 启动时优先选择 `initial_character_id`；若该 ID 不存在，则选择角色列表第一项；没有角色时保持空选择并禁用编辑、保存和导出控件。
- 选择已有角色后调用 `studio.open_character`，载入其工作区草稿并切换到“基础信息”页。
- 新建角色成功后，将草稿角色作为临时选项加入下拉菜单并立即选中。保存成功后，以后端返回的正式角色列表替换临时选项。

前端使用独立的 `editingCharacterId` 表示当前正在编辑的角色。`request.initial_character_id` 只表示主程序启动工作室时正在使用的角色，保存时继续原样传回，确保编辑其他角色不会切换桌宠角色。

如果表单有未保存修改，切换下拉项或点击“新建角色”前显示确认提示。用户取消时恢复原下拉值；确认后才打开或创建另一个角色。

## 导航与视觉一致性

工作室保留“基础信息”“人设卡”“立绘”“配色”四个导航项，并使用主设置页同样的内联 SVG 图标结构、图标尺寸、悬停状态和选中指示条。工作室侧栏宽度、品牌区、详情页头、设置组、输入控件、按钮和底栏的关键样式值与主设置页对应组件保持一致。

工作室仍保留少量领域专用样式，例如表情映射行和角色选择工具栏；这些样式建立在主设置页的主题变量及基础组件类之上。

## 配色编辑

后端请求通过 `THEME_COLOR_FIELDS` 提供和主设置页相同的字段 ID、中文标签、顺序及默认值。前端不再维护独立的 `themeLabels`。

配色页采用主设置页同款交互：

- 两列主题字段布局。
- 可点击的大色块和 `#RRGGBB` 文本输入。
- 点击色块打开包含 HSV 色板、色相条、HEX、RGB 和“取色/完成”操作的弹出编辑器。
- `studio.pick_screen_color` 复用 Python 层现有屏幕取色器。
- 合法颜色修改立即更新对应 `--sakura-*` CSS 变量，实时预览整个工作室。
- 非法文本显示错误状态，但不会写入 CSS 变量；保存仍由现有角色文档校验负责最终拒绝。

打开不同角色时，以该角色文档中的主题覆盖工作室主题。新建角色使用默认主题。切换角色时必须完整设置全部主题变量，避免上一个角色的颜色残留。

## 错误与状态处理

- 工作室 RPC 失败继续显示在底栏错误区域。
- 角色打开失败时恢复下拉菜单到当前文档对应项。
- 忙碌期间禁用角色下拉菜单、新建、保存、导出和立绘导入，避免并发操作。
- 屏幕取色取消不报错；真正失败时显示可读错误并保留原颜色。
- 新建角色对话框取消后不改变当前文档或下拉选择。

## 测试与验证

自动测试覆盖：

- 设置页只存在一个“修改角色”按钮，并且按钮传递当前下拉选择。
- 工作室不再包含角色列表页、搜索框和刷新按钮。
- 工作室包含持久角色下拉菜单和“新建角色”按钮。
- 工作室请求中的主题字段与 `THEME_COLOR_FIELDS` 完全一致。
- `studio.pick_screen_color` RPC 返回颜色、取消状态和错误。
- 前端包含主设置同款配色编辑器结构、主题变量更新和未保存切换保护。
- 现有角色打开、创建、保存、导出和立绘导入测试继续通过。

实现完成后先运行相关 Python 测试，再运行完整 `python -m pytest`，并运行两个 Tauri crate 的 Rust 测试。视觉验证至少覆盖当前角色启动、角色切换、新建草稿、四个导航页和配色弹窗。
