# 更新日志

## 0.9.9-dev - 开发中

这一节记录 `dev` 分支已经合入、准备进入 0.9.9 的改动。开发版仍可能调整配置和界面，不建议拿它替代日常使用的稳定包。

### 设置与首次启动

- 设置页改用 Tauri 界面，模型、角色、TTS、插件和系统选项集中在同一个窗口中。
- API 配置现在可以保存多套供应商，并分别给聊天、视觉和记忆整理选择模型。没有单独指定时，视觉和记忆任务会沿用聊天模型。
- 首次启动会引导选择或导入角色，再完成供应商和模型配置。已有配置不会重复进入引导。
- 主动截图可以选择分辨率。多显示器混合缩放下的截图坐标和尺寸也做了修正。

### 手机网页端

- 新增可选的 `sakura_mobile` 插件。手机浏览器可以查看当前角色的聊天记录，发送文字或图片，并把回复同步回桌面端。
- 手机端复用桌面的模型、记忆和当前角色，不会另起一套会话。为避免远程误操作，它不开放宿主工具，也不能在网页里切换角色。
- 网页和接口使用访问 token。局域网使用时不要把端口映射到公网；需要远程访问时，建议配合 Tailscale。

### 更新与发布

- Windows 包新增 `update.bat`。它会检查更新清单和 SHA-256，覆盖程序文件时保留 `data/`、角色、运行环境与用户插件配置。
- 发布包进一步缩小体积，工作流补上更新包、删除清单和校验文件的生成步骤。

### 稳定性修复

- 统一运行日志，减少控制台重复输出，并补齐模型请求、工具调用和后台任务的关键信息。
- 修复手机端忙碌状态、聊天回溯、记忆复用和模型切换后的配置刷新问题。
- 加固角色包导入导出、历史记录读取、存储迁移、TTS 播放和后台任务退出流程。
- Playwright 浏览器、MCP、工具参数校验和网络请求增加了更严格的边界检查。

## 0.9.8 - 2026-06-21

### 本地快速接话

等完整回复的时候，Sakura 不用再沉默了。她会先接一句符合当前语气的短回应——比如你说了一长串，她先回个"嗯，我在听"之类的。可以在设置里调延迟、概率和语音，不喜欢直接关掉也行。

### 长期记忆升级

之前记忆是一锅粥，现在分成了几层：核心信息、经历、任务、操作习惯，各自归类。聊天的时候 Sakura 会自动翻出相关内容，也会在对话中悄悄更新记忆。重复和冲突的信息她会自己整理。

### 插件系统

插件系统重写了。现在插件可以注册工具、设置页、事件和渲染器，插件自己的设置也移到了独立窗口。管理页上能看到每个插件的版本、权限和状态。Playwright 浏览器插件继续带着。

> 旧版插件如果用的是 `sdk.*` 接口，升级后可能用不了，需要插件作者按新规范适配。

### 角色工坊

Windows 用户双击 `start_studio.bat` 就能打开。这是个图形界面的角色编辑器：新建角色、写人格卡、搭主题配色、拖立绘和参考音频进去，最后导出 `.char` 文件，别人也能导入用。

### 其他

- 截图在高 DPI、多显示器和 Retina 下的错位和模糊修了。
- 点击桌宠就能叫出输入栏，不用每次都去找按钮。设置、历史和日志窗口现在有自己的任务栏图标，也不会被置顶的桌宠压住。
- 意外关掉重开，Sakura 会记得上一段聊了什么，不至于跟失忆一样。
- TTS 重做了底层，切换设置、退出程序和本地语音断开不会再崩。
- `0.9.8` 发布包不包含 `data/`，直接覆盖升级不会吃掉你的配置和聊天记录。
